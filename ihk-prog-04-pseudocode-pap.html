Pseudocode — Konventionen & Beispiel
Definition
Pseudocode ist eine informelle, sprachunabhängige Beschreibung eines Algorithmus in strukturierter Textform. Kein konkreter Syntax einer Programmiersprache — aber klare Struktur.
IHK akzeptiert verschiedene Pseudocode-Stile — wichtig ist Eindeutigkeit und Struktur!
Pseudocode-Schlüsselwörter (IHK-üblich)
ALGORITHMUS Name(Parameter) EINGABE: Beschreibung AUSGABE: Beschreibung WENN Bedingung DANN Anweisungen SONST Anweisungen ENDE WENN SOLANGE Bedingung TUE Anweisungen ENDE SOLANGE FÜR i VON 1 BIS n TUE Anweisungen ENDE FÜR ZURÜCKGEBEN Wert ENDE ALGORITHMUS
Beispiel: Maximum zweier Zahlen
ALGORITHMUS Maximum(a, b) EINGABE: Zwei Zahlen a, b AUSGABE: Die größere Zahl WENN a > b DANN ZURÜCKGEBEN a SONST ZURÜCKGEBEN b ENDE WENN ENDE ALGORITHMUS
Beispiel: Summe 1 bis n
ALGORITHMUS Summe(n) summe ← 0 FÜR i VON 1 BIS n TUE summe ← summe + i ENDE FÜR ZURÜCKGEBEN summe ENDE ALGORITHMUS // Trace n=3: summe=0→1→3→6
Eselsbrücke: Pseudocode schreiben
Einrücken = Struktur zeigen · WENN…ENDE WENN immer schließen · ← für Zuweisung · Variablennamen auf Deutsch OK · kein Semikolon nötig
Struktogramm (Nassi-Shneiderman)
Definition
Das Struktogramm (Nassi-Shneiderman-Diagramm) stellt Algorithmen als ineinander geschachtelte Blöcke dar. Kein GOTO möglich — erzwingt strukturierte Programmierung.
Entwickelt 1972 von Nassi & Shneiderman · DIN 66261
Struktogramm-Bausteine
1. Sequenz (Anweisung)
summe ← 0
i ← 1
Ausgabe summe
2. Selektion (Wenn-Dann-Sonst)
Bedingung: a > b ?
WAHR
FALSCH
max ← a
max ← b
3. Iteration (Schleife)
SOLANGE i ≤ n
summe ← summe + i
i ← i + 1
4. Fußgesteuerte Schleife (DO-WHILE)
Anweisung
WIEDERHOLE BIS Bedingung erfüllt
Unterschied kopf- vs. fußgesteuert
Kopfgesteuert Bedingung wird VOR dem ersten Durchlauf geprüft → 0 Durchläufe möglich (WHILE)
Fußgesteuert Bedingung NACH dem ersten Durchlauf → mindestens 1 Durchlauf (DO-WHILE)
Eselsbrücke: Kopf vs. Fuß
Kopfgesteuert = Kontrolle zuerst (wie Türsteher: erst prüfen, dann rein)
Fußgesteuert = mindestens einmal rein (erst testen, dann entscheiden)

Lernfeld: Daten systemübergreifend bereitstellen
DDL — Data Definition Language
Definition DDL
DDL-Befehle definieren und verändern die Datenbankstruktur (Tabellen, Spalten, Constraints). Sie wirken auf das Schema, nicht auf die Daten selbst. DDL-Befehle werden automatisch committed — kein ROLLBACK möglich!
Befehle: CREATE · ALTER · DROP · TRUNCATE
CREATE TABLE
CREATE TABLE Kunde ( KundeID INT PRIMARY KEY AUTO_INCREMENT, Name VARCHAR(100) NOT NULL, Email VARCHAR(150) UNIQUE, Gebdat DATE, Guthaben DECIMAL(10,2) DEFAULT 0.00 );
CREATE TABLE mit FOREIGN KEY
CREATE TABLE Bestellung ( BestellID INT PRIMARY KEY AUTO_INCREMENT, KundeID INT NOT NULL, Datum DATE, FOREIGN KEY (KundeID) REFERENCES Kunde(KundeID) ON DELETE CASCADE ON UPDATE CASCADE );
ON DELETE CASCADE: Bestellungen werden gelöscht, wenn Kunde gelöscht wird.
ALTER TABLE — Tabelle ändern
-- Spalte hinzufügen ALTER TABLE Kunde ADD Telefon VARCHAR(20); -- Spalte ändern ALTER TABLE Kunde MODIFY Name VARCHAR(200) NOT NULL; -- Spalte löschen ALTER TABLE Kunde DROP COLUMN Telefon;
DROP vs. TRUNCATE
DROP TABLE löscht Tabelle komplett inkl. Struktur
TRUNCATE löscht alle Zeilen, behält Struktur
DELETE (DML!) löscht gezielt mit WHERE
Eselsbrücke: DROP vs. TRUNCATE vs. DELETE
DROP = Das Haus abreißen (alles weg)
TRUNCATE = Tapete abreißen (Inhalt weg, Haus bleibt)
DELETE = Den Schrank räumen (gezielt, mit WHERE)
DML — Data Manipulation Language
Definition DML
DML-Befehle manipulieren die Daten innerhalb bestehender Tabellen. Sie können mit ROLLBACK rückgängig gemacht werden (innerhalb einer Transaktion).
Befehle: INSERT · UPDATE · DELETE
INSERT — Daten einfügen
-- Einzelner Datensatz INSERT INTO Kunde (Name, Email, Gebdat) VALUES ('Max Müller', 'max@mail.de', '1990-05-15'); -- Mehrere Datensätze auf einmal INSERT INTO Kunde (Name, Email) VALUES ('Erika Meier', 'erika@mail.de'), ('Tom Schmidt', 'tom@mail.de');
⚠️ Spaltenliste immer angeben — schützt vor Spaltenreihenfolgefehlern!
UPDATE — Daten ändern
-- Mit WHERE (immer angeben!) UPDATE Kunde SET Email = 'neu@mail.de', Guthaben = Guthaben + 50 WHERE KundeID = 1;
⚠️ UPDATE ohne WHERE ändert ALLE Zeilen — klassischer Prüfungsfehler!
DELETE — Daten löschen
-- Einzelne Zeile löschen DELETE FROM Kunde WHERE KundeID = 5; -- Mit Unterabfrage DELETE FROM Bestellung WHERE KundeID IN ( SELECT KundeID FROM Kunde WHERE Guthaben < 0 );
⚠️ DELETE ohne WHERE löscht ALLE Zeilen!
Eselsbrücke: DML-Gefahr
„Update und Delete ohne WHERE —
Unfall garantiert, Daten weg!"
→ Immer WHERE mitschreiben!
DCL — Data Control Language
Definition DCL
DCL-Befehle steuern die Zugriffsrechte auf Datenbankobjekte. Sie regeln, welcher Benutzer welche Operationen ausführen darf.
Befehle: GRANT · REVOKE
GRANT — Rechte vergeben
-- SELECT-Recht auf Tabelle vergeben GRANT SELECT ON Kunde TO 'benutzer1'; -- Mehrere Rechte GRANT SELECT, INSERT, UPDATE ON Bestellung TO 'sachbearbeiter'; -- Alle Rechte GRANT ALL PRIVILEGES ON *.* TO 'admin';
REVOKE — Rechte entziehen
REVOKE SELECT ON Kunde FROM 'benutzer1';
Rechtekategorien (IHK)
SELECT Lesen
INSERT Einfügen
UPDATE Ändern
DELETE Löschen
ALL PRIVILEGES Alle Rechte
Eselsbrücke: GRANT vs. REVOKE
GRANT = Gibt Rechte (Grün)
REVOKE = Raubt Rechte (Rot)
JOINs — Tabellen verknüpfen
Definition JOIN
Ein JOIN verknüpft zwei oder mehr Tabellen anhand einer gemeinsamen Spalte (meist PK ↔ FK) und gibt eine kombinierte Ergebnismenge zurück.
Beispieldaten: Tabelle Kunde (KundeID, Name) + Tabelle Bestellung (BestellID, KundeID, Produkt)
INNER JOIN (Standard-JOIN)
Gibt nur Zeilen zurück, die in beiden Tabellen einen übereinstimmenden Wert haben.
SELECT k.Name, b.Produkt FROM Kunde k INNER JOIN Bestellung b ON k.KundeID = b.KundeID;
Kunden ohne Bestellung erscheinen nicht im Ergebnis.
LEFT JOIN (LEFT OUTER JOIN)
Gibt alle Zeilen der linken Tabelle zurück, auch wenn kein Treffer in der rechten Tabelle vorhanden ist. Fehlende Werte werden mit NULL gefüllt.
SELECT k.Name, b.Produkt FROM Kunde k LEFT JOIN Bestellung b ON k.KundeID = b.KundeID;
Kunden ohne Bestellung erscheinen mit NULL bei Produkt.
RIGHT JOIN (RIGHT OUTER JOIN)
Gibt alle Zeilen der rechten Tabelle zurück, auch wenn kein Treffer links vorhanden ist.
SELECT k.Name, b.Produkt FROM Kunde k RIGHT JOIN Bestellung b ON k.KundeID = b.KundeID;
Selten genutzt — meist durch Tauschen der Tabellen als LEFT JOIN lösbar.
FULL OUTER JOIN
Gibt alle Zeilen beider Tabellen zurück, unabhängig ob ein Treffer vorhanden ist. Fehlende Werte werden mit NULL aufgefüllt.
In MySQL nicht direkt vorhanden → mit UNION aus LEFT + RIGHT JOIN simulieren.
CROSS JOIN
Liefert das kartesische Produkt — jede Zeile der linken Tabelle mit jeder Zeile der rechten (ohne ON-Bedingung).
3 Kunden × 4 Produkte = 12 Zeilen im Ergebnis.
JOIN-Übersicht & Visualisierung
Ergebnisvergleich (Beispieldaten)
Kunde: Müller(1), Meier(2), Schmidt(3·keine Bestellung)
Bestellung: B1→Müller, B2→Müller, B3→Meier, B4→Unbekannt(FK=99)

Eselsbrücke: JOIN-Typen
„Inner = Intersection (Schnittmenge)"
„Left = Linke Tabelle komplett"
„Right = Rechte Tabelle komplett"
„Full = Fülle von beiden Seiten"

Mengenalgebra: JOIN = ∩ · LEFT = A∪(A∩B) · RIGHT = B∪(A∩B)
JOIN mit Alias & mehreren Bedingungen
-- Alias für Lesbarkeit SELECT k.Name AS Kunde, b.Produkt AS Artikel, b.Preis * b.Menge AS Wert FROM Kunde k INNER JOIN Bestellung b ON k.KundeID = b.KundeID WHERE b.Preis > 100 ORDER BY Wert DESC;
3-Tabellen-JOIN
SELECT k.Name, p.Kategorie, b.Menge FROM Kunde k JOIN Bestellung b ON k.KundeID = b.KundeID JOIN Produkt p ON b.ProduktID = p.ProduktID;
Jede weitere Tabelle braucht eine eigene JOIN … ON Zeile.

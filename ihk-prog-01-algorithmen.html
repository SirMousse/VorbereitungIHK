Was ist ein Algorithmus? — Definition
Algorithmus (IHK-Definition)
Ein Algorithmus ist eine eindeutige, endliche Folge von Anweisungen zur Lösung eines Problems oder einer Aufgabenklasse. Er muss folgende Eigenschaften erfüllen:
Eigenschaften (Pflichtmerkmale)
Finitheit endliche Anzahl von Schritten
Definitheit jeder Schritt eindeutig beschrieben
Terminiertheit endet nach endlicher Zeit
Effektivität jeder Schritt ausführbar
Determiniertheit gleiche Eingabe → gleiche Ausgabe
Korrektheit liefert das richtige Ergebnis
Eselsbrücke: Algorithmus-Eigenschaften
„Finde Den Tollsten Effektiven Deterministischen Korrekten"
Finitheit · Definitheit · Terminiertheit · Effektivität · Determiniertheit · Korrektheit

Lineare Suche (Linear Search)
Definition & Wann einsetzen
Die lineare Suche durchsucht eine Liste Element für Element von vorne nach hinten, bis das gesuchte Element gefunden oder die Liste erschöpft ist.
Wann: unsortierte Listen · kleine Datenmengen · einfachste Suche
Komplexität
BestO(1)
AverageO(n/2)
WorstO(n)
Trace-Tabelle: Suche nach 7 in [3,7,1,9,4]
SchrittIndexWertGefunden?
103Nein
217✓ JA
Code
Python
def lineare_suche(liste, ziel): for i, wert in enumerate(liste): if wert == ziel: return i # Index zurückgeben return -1 # nicht gefunden
Java
int lineareSuche(int[] arr, int ziel) { for (int i = 0; i < arr.length; i++) { if (arr[i] == ziel) return i; } return -1; }
Eselsbrücke
Linear = Langsam aber Lässig — jede Liste, egal ob sortiert.
Binäre Suche (Binary Search)
Definition & Wann einsetzen
Die binäre Suche halbiert bei jedem Schritt den Suchbereich. Sie vergleicht den gesuchten Wert mit dem mittleren Element und verwirft die irrelevante Hälfte.
Wann: NUR auf sortierten Listen! · große Datenmengen · sehr effizient
Komplexität
BestO(1)
AverageO(log n)
WorstO(log n)
Trace: Suche 7 in [1,3,4,7,9,12,15]
SchrittLinksRechtsMitteWert
10637 ✓
1 Schritt statt 4 bei linearer Suche!
Code
Python
def binaere_suche(liste, ziel): li, re = 0, len(liste) - 1 while li <= re: mitte = (li + re) // 2 if liste[mitte] == ziel: return mitte elif liste[mitte] < ziel: li = mitte + 1 else: re = mitte - 1 return -1
Java
int binaereSuche(int[] arr, int ziel) { int li=0, re=arr.length-1; while (li<=re) { int m=(li+re)/2; if (arr[m]==ziel) return m; else if (arr[m]1; else re=m-1; } return -1; }
Eselsbrücke
Binär = Buch aufschlagen in der Mitte — links oder rechts weitersuchen. Voraussetzung: sortiert!
Bubble Sort
Definition & Wann einsetzen
Bubble Sort vergleicht benachbarte Elemente und tauscht sie, wenn sie in falscher Reihenfolge sind. Große Elemente „blubbern" nach oben.
Wann: Lernzwecke · sehr kleine Listen · einfachster Sortieralgorithmus
BestO(n)
AverageO(n²)
WorstO(n²)
Trace: [5, 3, 8, 1] sortieren
PassListeTausch
Start[5, 3, 8, 1]—
1[3, 5, 1, 8]5↔3, 8↔1
2[3, 1, 5, 8]5↔1
3[1, 3, 5, 8]3↔1
4[1, 3, 5, 8]✓ Fertig
Python
def bubble_sort(lst): n = len(lst) for i in range(n): for j in range(0, n-i-1): if lst[j] > lst[j+1]: lst[j],lst[j+1] = lst[j+1],lst[j] return lst
Java
void bubbleSort(int[] a) { int n=a.length; for(int i=0;i<n;i++) for(int j=0;j<n-i-1;j++) if(a[j]>a[j+1]) { int t=a[j]; a[j]=a[j+1]; a[j+1]=t; } }
Eselsbrücke
Blasen steigen auf — große Zahlen blubbern nach rechts. n² = n·n Vergleiche → langsam bei großen Listen.
Selection Sort
Definition & Wann einsetzen
Selection Sort sucht in jedem Durchlauf das kleinste Element im unsortierten Teil und stellt es an die richtige Position.
Wann: kleine Listen · wenig Speicher nötig · Lernzwecke
BestO(n²)
AverageO(n²)
WorstO(n²)
Trace: [5, 3, 8, 1]
PassMin-PosListe
1idx=3 (Wert 1)[1, 3, 8, 5]
2idx=1 (Wert 3)[1, 3, 8, 5]
3idx=3 (Wert 5)[1, 3, 5, 8]
Python
def selection_sort(lst): n = len(lst) for i in range(n): min_idx = i for j in range(i+1, n): if lst[j] < lst[min_idx]: min_idx = j lst[i],lst[min_idx] = lst[min_idx],lst[i] return lst
Eselsbrücke
Selection = Suche immer das Skleinste und stelle es vorne hin.
Insertion Sort & Merge Sort
Insertion Sort — Definition
Nimmt ein Element und fügt es an der richtigen Stelle in den bereits sortierten Teil ein — wie Karten sortieren in der Hand.
Wann: fast sortierte Listen · kleine Datenmengen · stabil
BestO(n)
AvgO(n²)
WorstO(n²)
Python
def insertion_sort(lst): for i in range(1, len(lst)): key = lst[i] j = i - 1 while j >= 0 and lst[j] > key: lst[j+1] = lst[j] j -= 1 lst[j+1] = key return lst
Eselsbrücke: Insertion Sort
Insertion = In die richtige Lücke einfügen — wie Spielkarten in der Hand sortieren.
Merge Sort — Definition
Merge Sort teilt die Liste rekursiv in Hälften, sortiert jede Hälfte und verschmilzt (merged) die sortierten Teile. Prinzip: Divide & Conquer.
Wann: große Datenmengen · stabile Sortierung nötig · effizient
BestO(n log n)
AvgO(n log n)
WorstO(n log n)
Ablauf: [5,3,8,1] → Merge Sort
[5,3,8,1]
[5,3]   [8,1]
[5][3] [8][1]
[3,5]   [1,8]
[1,3,5,8] ✓
Eselsbrücke: Merge Sort
Merge = Mischen nach dem Teilen — „Teile und herrsche" (Divide & Conquer)

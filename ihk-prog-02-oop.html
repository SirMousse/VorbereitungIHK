OOP — Grundbegriffe & Die 4 Säulen
Objektorientierte Programmierung (OOP)
OOP ist ein Programmierparadigma, bei dem ein Programm aus interagierenden Objekten besteht, die Daten (Attribute) und Verhalten (Methoden) kapseln.
Klasse vs. Objekt
Klasse = Bauplan / Vorlage (abstrakt)
Objekt = Instanz einer Klasse (konkret)
Instanziierung = Erstellen eines Objekts aus Klasse
Klasse Auto → Objekte: meinAuto, deinAuto (beide sind Autos)
Die 4 Säulen der OOP (IHK!)
1. Kapselung Daten verstecken + Zugriff kontrollieren
2. Vererbung Eigenschaften von Klasse übernehmen
3. Polymorphismus Gleicher Name, verschiedenes Verhalten
4. Abstraktion Komplexität verstecken, Wesentliches zeigen

Klassen & Objekte — Code
Python
class Auto: # Konstruktor def __init__(self, marke, ps): self.marke = marke # Attribut self.ps = ps self.__km = 0 # privat (__) def fahren(self, km): # Methode self.__km += km print(f"Gefahren: {km} km") def get_km(self): # Getter return self.__km # Instanziierung mein_auto = Auto("BMW", 200) mein_auto.fahren(100)
Java
public class Auto { private String marke; private int ps; private int km = 0; public Auto(String marke, int ps) { this.marke = marke; this.ps = ps; } public void fahren(int km) { this.km += km; } public int getKm() { return km; } }
Kapselung & Zugriffsmodifikatoren
Kapselung (Encapsulation)
Kapselung bedeutet, interne Daten einer Klasse zu verbergen und nur über definierte Methoden (Getter/Setter) zugänglich zu machen. Schützt vor unkontrolliertem Zugriff.
Prinzip: Attribute privat, Methoden öffentlich

Getter & Setter
Java
private int alter; public int getAlter() { return alter; } public void setAlter(int alter) { if (alter >= 0) this.alter = alter; // Validierung im Setter! }
Eselsbrücke: Kapselung
Kapselung = Koffer mit Schloss — du weißt was drin ist, kommst aber nur über den Kofer-Handler (Getter/Setter) ran.
Vererbung (Inheritance)
Definition
Vererbung ermöglicht einer Klasse (Unterklasse/Kindklasse), Attribute und Methoden einer anderen Klasse (Oberklasse/Elternklasse) zu übernehmen und zu erweitern.
Prinzip: „is-a"-Beziehung · Elektroauto IS-A Auto
Python
class Fahrzeug: # Oberklasse def __init__(self, marke): self.marke = marke def starten(self): print("Fahrzeug startet") class Elektroauto(Fahrzeug): # erbt def __init__(self, marke, akku): super().__init__(marke) # Eltern self.akku = akku def starten(self): # überschreiben print("Lautlos starten")
Java
class Elektroauto extends Fahrzeug { private int akku; public Elektroauto(String m, int a){ super(m); // Elternkonstruktor this.akku = a; } @Override public void starten() { System.out.println("Lautlos"); } }
Eselsbrücke: Vererbung
„Elektroauto IS-A Fahrzeug" — erbt alles vom Elternteil und kann eigenes hinzufügen. super() = Elternteil anrufen.
Polymorphismus & Abstraktion
Polymorphismus
Polymorphismus („Vielgestaltigkeit") bedeutet, dass eine Methode mit gleichem Namen in verschiedenen Klassen unterschiedlich implementiert werden kann.
Überschreiben (Override) Methode in Unterklasse neu implementieren (Laufzeit)
Überladen (Overload) Gleicher Methodenname, verschiedene Parameter (Kompilierzeit, Java)
Python — Polymorphismus
class Tier: def laut(self): print("...") class Hund(Tier): def laut(self): print("Wuff") class Katze(Tier): def laut(self): print("Miau") for tier in [Hund(), Katze()]: tier.laut() # Wuff / Miau
Eselsbrücke: Polymorphismus
Poly = viele · Morphismus = Formen. Gleicher Befehl (laut()), verschiedene Reaktion — wie „Hallo" in verschiedenen Sprachen.
Abstraktion
Abstraktion versteckt unnötige Details und zeigt nur das Wesentliche. Abstrakte Klassen/Interfaces definieren einen Vertrag ohne Implementierung.
Java — abstrakte Klasse
abstract class Form { abstract double flaeche(); // kein Body void drucken() { System.out.println(flaeche()); } } class Kreis extends Form { double r; double flaeche() { return Math.PI*r*r; } }
Eselsbrücke: Abstraktion
Abstrakte Klasse = Bauplan mit Lücken. Die Unterklasse füllt die Lücken aus. Kann nicht direkt instanziiert werden!
Interface & Abstrakte Klasse
Interface (Java) / ABC (Python)
Ein Interface definiert einen Vertrag — eine Menge von Methoden, die eine Klasse implementieren muss. Enthält keine Implementierung (nur Signaturen).
Interface = „kann-machen"-Beziehung · Klasse implements Interface
Java — Interface
interface Fahrbar { void fahren(int km); // kein Body int getTempo(); } class Auto implements Fahrbar { public void fahren(int km) { /*...*/ } public int getTempo() { return 120; } } // Eine Klasse kann mehrere Interfaces // implementieren — keine Mehrfachvererbung!

Eselsbrücke: Interface vs. Abstrakte Klasse
Interface = Steckdosenstandard — jedes Gerät das den Standard erfüllt, passt.
Abstrakte Klasse = Halbfertiges Produkt — Unterklasse ergänzt den Rest.
Wichtige OOP-Konzepte
Statische Methoden & Attribute
static gehört zur Klasse, nicht zum Objekt — kein self/this nötig, existiert einmal für alle Instanzen.
Java
static int anzahl = 0; // Klassenvar. static int getAnzahl() { return anzahl; }
Konstruktor-Überladen (Java)
class Punkt { int x, y; public Punkt() { x=0; y=0; } public Punkt(int x,int y){ this.x=x; this.y=y; } }
Aggregation vs. Komposition
Aggregation „hat-ein" · unabhängig · Motor kann ohne Auto existieren
Komposition „besteht aus" · abhängig · Raum existiert nicht ohne Haus

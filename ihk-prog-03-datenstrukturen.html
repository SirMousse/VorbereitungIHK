Array & Liste — Grundlagen
Array (Feld)
Eine feste Sammlung von Elementen gleichen Typs mit direktem Indexzugriff. Größe wird bei der Erstellung festgelegt.
Index beginnt bei 0 · Zugriff per arr[i] in O(1)
Java
int[] zahlen = new int[5]; zahlen[0] = 10; int[] vorbel = {1, 2, 3, 4, 5}; System.out.println(vorbel.length); // 5
Liste (dynamisches Array)
Eine dynamische Sammlung, die wächst und schrumpft. In Python: list, in Java: ArrayList.
Python
lst = [1, 2, 3] lst.append(4) # hinzufügen lst.insert(1, 99) # an Position 1 lst.remove(2) # Wert 2 entfernen lst.pop() # letztes entfernen len(lst) # Länge lst[0:2] # Slicing
Java
ArrayList<Integer> lst = new ArrayList<>(); lst.add(1); lst.add(0, 99); // an Index 0 lst.remove(0); // Index entfernen lst.size(); // Länge lst.get(1); // Zugriff
Eselsbrücke: Array vs. Liste
Array = Schubladenschrank — feste Anzahl Schubladen, direkter Griff.
Liste = Einkaufskorb — beliebig viele Elemente rein und raus.
Zugriff & Komplexität Vergleich
OperationArrayLinkedListArrayList
Zugriff [i]O(1)O(n)O(1)
Einfügen EndeO(1)*O(1)O(1)*
Einfügen MitteO(n)O(1)O(n)
SuchenO(n)O(n)O(n)
Löschen MitteO(n)O(1)O(n)
* amortisiert · LinkedList gut bei häufigem Einfügen/Löschen in der Mitte
Wann was verwenden?
Array Feste Größe, viele Index-Zugriffe
ArrayList Variable Größe, viele Zugriffe per Index
LinkedList Häufiges Einfügen/Löschen in der Mitte

Stack (Stapel) — LIFO
Definition
Ein Stack (Stapel) ist eine Datenstruktur nach dem LIFO-Prinzip (Last In, First Out) — das zuletzt hinzugefügte Element wird zuerst entnommen.
Wann: Rückgängig-Funktion (Undo) · Rekursions-Stack · Browser-Verlauf · Klammerprüfung
Visualisierung
10
20
30 ← TOP (push/pop hier)
push(30) → auf Stapel · pop() → 30 entnehmen
Operationen
push(x) Element oben drauflegen · O(1)
pop() Oberstes Element entnehmen · O(1)
peek() Oberstes Element ansehen (nicht entnehmen)
isEmpty() Prüfen ob leer
Python
stack = [] stack.append(10) # push stack.append(20) top = stack.pop() # pop → 20 peek = stack[-1] # peek → 10 leer = len(stack) == 0
Java
Stack<Integer> s = new Stack<>(); s.push(10); s.push(20); int top = s.pop(); // 20 int peek = s.peek(); // 10 boolean leer = s.isEmpty();
Eselsbrücke: Stack LIFO
LIFO = Letzter In, First Out — wie ein Tellerstapel: letzter drauf = erster runter.
Queue (Warteschlange) — FIFO
Definition
Eine Queue (Warteschlange) folgt dem FIFO-Prinzip (First In, First Out) — das zuerst hinzugefügte Element wird zuerst entnommen.
Wann: Druckerwarteschlange · Aufgabenplanung · Breitensuche (BFS) · Netzwerkpakete
Visualisierung
dequeue→
10
20
30
←enqueue
Operationen
enqueue(x) Element hinten anhängen · O(1)
dequeue() Vorderstes Element entnehmen · O(1)
peek() Vorderstes ansehen
isEmpty() Prüfen ob leer
Python
from collections import deque q = deque() q.append(10) # enqueue q.append(20) first = q.popleft() # dequeue → 10
Java
Queue<Integer> q = new LinkedList<>(); q.offer(10); // enqueue q.offer(20); int f = q.poll(); // dequeue → 10 int p = q.peek(); // ansehen → 20
Eselsbrücke: Queue FIFO
FIFO = First In, First Out — wie eine Supermarktkasse: wer zuerst ansteht, zahlt zuerst.
Dictionary / HashMap (Schlüssel-Wert)
Definition
Ein Dictionary (Python) / HashMap (Java) speichert Daten als Schlüssel-Wert-Paare. Der Zugriff erfolgt über den Schlüssel in O(1) — nicht über einen Index.
Wann: schnelle Suche nach Schlüssel · Lookup-Tabellen · Zählen von Häufigkeiten
Python
person = { "name": "Max", "alter": 25, "stadt": "Berlin" } person["email"] = "max@mail.de" # add del person["stadt"] # delete "name" in person # exists? for k, v in person.items(): # iterate print(k, v)
Java
HashMap<String,Integer> map = new HashMap<>(); map.put("Max", 25); map.get("Max"); // → 25 map.remove("Max"); map.containsKey("Max"); // exists? for (String k : map.keySet()) System.out.println(k + map.get(k));
Eselsbrücke: Dictionary
Dict = Wörterbuch — du kennst das Wort (Key), schlägst nach und findest die Bedeutung (Value). Kein Blättern nötig — direkt!
Set (Menge)
Speichert eindeutige Elemente ohne Reihenfolge. Perfekt zum Entfernen von Duplikaten und Mengenoperationen.
Python
s = {1, 2, 3} s.add(4) s.discard(2) a & b # Schnittmenge a | b # Vereinigung a - b # Differenz
Baum & Rekursion
Binärbaum
Ein Baum besteht aus Knoten, die über Kanten verbunden sind. Jeder Knoten hat max. einen Elternknoten. Ein Binärbaum hat max. 2 Kindknoten pro Knoten.
5 ← Wurzel
/ \
3 8
/ \ \
1 4 9
Wann: Suchbäume (BST) · Heap · Dateisystem · Ausdrucksbaum
Rekursion
Rekursion = eine Funktion ruft sich selbst auf. Benötigt immer einen Basisfall (Abbruchbedingung) und einen Rekursionsfall.
Python — Fakultät rekursiv
def fakultaet(n): if n <= 1: # Basisfall return 1 return n * fakultaet(n-1) # Rekursion # Trace: fakultaet(4) # 4 * fak(3) → 4 * 3 * fak(2) # → 4 * 3 * 2 * fak(1) → 24
Eselsbrücke: Rekursion
Rekursion = Matrjoschka-Puppe — öffne Puppe, drin ist wieder eine Puppe... bis zur kleinsten (Basisfall).

Grundbegriffe & Definitionen (IHK)
Event (Ereignis)
Ein Ereignis ist eine Aktion oder ein Vorkommnis im Browser, das durch den Benutzer oder das System ausgelöst wird und auf das eine Webanwendung reagieren kann.
Beispiele: Mausklick, Tastatureingabe, Seitenladung, Formularübermittlung
Event Listener (Ereignishorcher)
Ein Event Listener ist eine Funktion oder ein Objekt, das auf einem DOM-Element registriert wird und auf ein bestimmtes Ereignis wartet. Er „horcht" auf das Eintreten des Events.
Registrierung via: element.addEventListener(typ, handler)
Event Handler (Ereignisbehandler)
Ein Event Handler ist die Funktion, die aufgerufen wird, sobald das überwachte Ereignis eintritt. Sie enthält die Logik zur Reaktion auf das Event.
Der Handler erhält automatisch das Event-Objekt als Parameter.
Event-Objekt
Das Event-Objekt wird beim Eintreten eines Ereignisses automatisch erzeugt und an den Handler übergeben. Es enthält alle Informationen über das Ereignis.
Wichtige Properties: target, type, currentTarget
DOM (Document Object Model)
Das DOM ist eine baumartige Programmierschnittstelle, die HTML-Dokumente als Objekte repräsentiert. Events werden auf DOM-Elementen (Knoten) registriert und verarbeitet.

Event-Propagation (Ereignisweiterleitung)
Definition Propagation
Unter Event-Propagation versteht man die Weitergabe eines Ereignisses durch den DOM-Baum. Sie läuft in drei Phasen ab:
Phase 1
Capturing
window → target
von oben nach unten
→
Phase 2
Target
Ziel-Element
Event trifft ein
→
Phase 3
Bubbling
target → window
von unten nach oben
Bubbling
Nach dem Eintreten wandert das Event vom Ziel-Element aufwärts durch alle Eltern-Elemente bis zum window. Standard-Verhalten der meisten Events.
Capturing
Das Event wandert zuerst vom window abwärts zum Ziel. Nur aktiv wenn { capture: true } gesetzt ist.
stopPropagation()
Unterbricht die Weitergabe des Events an übergeordnete oder untergeordnete Elemente. Das Event wird nicht weiter weitergeleitet.
e.stopPropagation(); // kein Bubbling mehr
preventDefault()
Verhindert das Standardverhalten des Browsers für dieses Event (z. B. Seitenreload bei Formular-Submit, Link-Navigation).
e.preventDefault(); // kein Seitenreload
Event Delegation
Definition
Event Delegation ist ein Entwurfsmuster, bei dem statt auf jedem Kind-Element ein einzelner Event Listener auf dem gemeinsamen Eltern-Element registriert wird. Das Bubbling-Prinzip leitet alle Events nach oben weiter.
Vorteile (IHK-relevant)
· Weniger Listener → bessere Performance
· Funktioniert auch für dynamisch hinzugefügte Elemente
· Weniger Speicherverbrauch
// Listener auf Parent statt auf jedem <li> ul.addEventListener('click', (e) => { if (e.target.tagName === 'LI') { console.log('Geklickt:', e.target.textContent); } });
e.target → auslösendes Element  |  e.currentTarget → Element mit Listener
Syntax & Methoden im Vergleich
Methode Syntax Bewertung
addEventListener el.addEventListener('click', fn) ✔ Empfohlen · mehrere Listener möglich · entfernbar
onclick-Attribut (HTML) <button onclick="fn()"> ✖ Vermeiden · HTML und JS vermischt (kein Separation of Concerns)
onclick-Property (JS) el.onclick = fn; ✖ Nur 1 Listener · überschreibt sich selbst
addEventListener — vollständige Syntax
element.addEventListener(type, handler, options) // type → Event-Typ als String: 'click', 'submit', … // handler → Funktion (benannt oder anonym) // options → { capture, once, passive } element.removeEventListener('click', handler); // ⚠ Nur mit benannter Funktion entfernbar!
Options-Parameter (IHK-Wissen)
capture: true Listener in Capturing-Phase aktiv
once: true     Listener feuert nur einmal
passive: true  preventDefault() wird nicht aufgerufen → bessere Scroll-Performance

// Vollständiges Beispiel (IHK-konform) const btn = document.querySelector('#mein-btn'); function handleClick(event) { event.preventDefault(); // Standardaktion stoppen console.log(event.target); // auslösendes Element console.log(event.type); // 'click' } btn.addEventListener('click', handleClick); // Entfernen (nur mit benannter Funktion!): btn.removeEventListener('click', handleClick);
Fachbegriff: Separation of Concerns
Separation of Concerns (Trennung der Zuständigkeiten) ist ein Entwurfsprinzip, nach dem HTML (Struktur), CSS (Darstellung) und JavaScript (Verhalten) strikt getrennt werden sollen.
→ Daher: Event Listener immer in JS registrieren, nicht als HTML-Attribut (onclick="...").
Merksatz für die Prüfung
Event Listener = Wächter · registriert sich auf einem Element
Event Handler = Ausführer · enthält die Reaktionslogik
Event-Objekt = Datenbote · liefert alle Infos zum Ereignis
Propagation = Reiseroute · Capturing → Target → Bubbling

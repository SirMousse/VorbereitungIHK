SELECT — Grundstruktur & Ausführungsreihenfolge
Vollständige SELECT-Syntax
SELECT Spalten / * / Ausdrücke FROM Tabelle(n) WHERE Zeilenbedingung GROUP BY Gruppierungsspalte(n) HAVING Bedingung auf Gruppen ORDER BY Spalte ASC|DESC LIMIT Anzahl Zeilen;

SELECT — Grundabfragen
Alle Spalten / bestimmte Spalten
SELECT * FROM Bestellung; SELECT Kunde, Produkt FROM Bestellung; SELECT DISTINCT Stadt FROM Bestellung; -- DISTINCT = keine Duplikate
Aliase (AS)
SELECT Kunde AS 'Kundenname', Preis * Menge AS Gesamtwert FROM Bestellung;
Alias gilt nur in SELECT und ORDER BY — nicht in WHERE!
Sortieren mit ORDER BY
SELECT * FROM Bestellung ORDER BY Preis DESC; -- absteigend ORDER BY Kunde ASC, Preis DESC; -- mehrere Kriterien möglich
Begrenzen mit LIMIT
SELECT * FROM Bestellung ORDER BY Preis DESC LIMIT 3; -- nur die 3 teuersten
WHERE — Filtern & Operatoren
Definition WHERE
WHERE filtert einzelne Zeilen vor der Gruppierung. Nur Zeilen, die die Bedingung erfüllen, werden berücksichtigt. WHERE kann keine Aggregatfunktionen enthalten (→ dafür HAVING).
Vergleichsoperatoren
WHERE Preis = 999 -- gleich WHERE Preis <> 999 -- ungleich (auch !=) WHERE Preis > 100 -- größer WHERE Preis <= 500 -- kleiner gleich
Logische Operatoren
WHERE Stadt = 'Berlin' AND Preis > 100 WHERE Stadt = 'Berlin' OR Stadt = 'Hamburg' WHERE NOT Stadt = 'München'
Spezialoperatoren
-- BETWEEN: Bereich (inklusiv) WHERE Preis BETWEEN 50 AND 500 -- IN: Liste von Werten WHERE Stadt IN ('Berlin', 'Hamburg') -- LIKE: Mustersuche WHERE Kunde LIKE 'M%' -- beginnt mit M WHERE Kunde LIKE '%er' -- endet mit er WHERE Kunde LIKE '_ü%' -- _ = genau 1 Zeichen -- NULL-Prüfung WHERE Preis IS NULL WHERE Preis IS NOT NULL
Eselsbrücke: LIKE-Platzhalter
% = beliebig viele Zeichen (Prozent = Plural)
_ = genau ein Zeichen (Unterstrich = ein Strich)
GROUP BY & HAVING
Definition GROUP BY
GROUP BY fasst Zeilen mit gleichen Werten in einer Spalte zu Gruppen zusammen. Auf Gruppen können Aggregatfunktionen angewendet werden.
Alle Spalten in SELECT, die keine Aggregatfunktion sind, müssen in GROUP BY stehen!
Definition HAVING
HAVING filtert Gruppen (nach GROUP BY) — genau wie WHERE Zeilen filtert. HAVING darf Aggregatfunktionen enthalten, WHERE nicht.
Eselsbrücke: WHERE vs. HAVING
WHERE = Zeilen filtern (vor Gruppierung)
HAVING = Gruppen filtern (nach Gruppierung)
„Was einzeln? → WHERE · Haufen? → HAVING"
-- Umsatz je Kunde (Summe Preis × Menge) SELECT Kunde, SUM(Preis * Menge) AS Umsatz, COUNT(*) AS Anzahl_Bestellungen FROM Bestellung GROUP BY Kunde HAVING SUM(Preis * Menge) > 500 ORDER BY Umsatz DESC;
Ergebnis: Nur Kunden mit Umsatz über 500 €, absteigend sortiert
Aggregatfunktionen
Definition
Aggregatfunktionen berechnen einen einzelnen Wert aus einer Menge von Zeilen (einer Gruppe oder der gesamten Tabelle). Sie ignorieren NULL-Werte (Ausnahme: COUNT(*)).
FunktionBeschreibungBeispiel
COUNT(*)Anzahl aller ZeilenCOUNT(*) → 5
COUNT(Spalte)Anzahl Nicht-NULL-WerteCOUNT(Preis) → 5
SUM(Spalte)Summe der WerteSUM(Preis) → 2451
AVG(Spalte)DurchschnittAVG(Preis) → 490.2
MIN(Spalte)Kleinster WertMIN(Preis) → 25.00
MAX(Spalte)Größter WertMAX(Preis) → 999.00
Eselsbrücke: Aggregatfunktionen
„Count Sums Average MinMax"
COUNT · SUM · AVG · MIN · MAX
Alle ignorieren NULL — außer COUNT(*)!
-- Alle Aggregatfunktionen auf Beispieldaten SELECT COUNT(*) AS Anzahl, -- 5 SUM(Preis * Menge) AS Gesamtums., -- 3079 AVG(Preis) AS Ø_Preis, -- 490.2 MIN(Preis) AS Min_Preis, -- 25.00 MAX(Preis) AS Max_Preis -- 999.00 FROM Bestellung;
Unterabfragen (Subqueries)
Definition
Eine Unterabfrage (Subquery) ist eine SELECT-Abfrage, die innerhalb einer anderen SQL-Abfrage steht. Das Ergebnis der inneren Abfrage wird von der äußeren verwendet.
Unterabfragen stehen in Klammern und werden zuerst ausgeführt.
Unterabfrage in WHERE
-- Alle Bestellungen teurer als Durchschnitt SELECT * FROM Bestellung WHERE Preis > ( SELECT AVG(Preis) FROM Bestellung );
Unterabfrage mit IN
-- Kunden, die einen Laptop bestellt haben SELECT DISTINCT Kunde FROM Bestellung WHERE Kunde IN ( SELECT Kunde FROM Bestellung WHERE Produkt = 'Laptop' );
Eselsbrücke: Subquery
Subquery = „Frage in der Frage"
Innen zuerst → Ergebnis nach außen
Immer in (runden Klammern)
String- & Datumsfunktionen
Wichtige Stringfunktionen
UPPER('hallo') → 'HALLO' LOWER('HALLO') → 'hallo' LENGTH('Hallo') → 5 CONCAT('Ha','llo') → 'Hallo' SUBSTRING('Hallo',1,3) → 'Hal' TRIM(' Hallo ') → 'Hallo' REPLACE('Hallo','l','r') → 'Harro'
Wichtige Datumsfunktionen
NOW() → 2024-06-01 10:30:00 CURDATE() → 2024-06-01 YEAR(Datum) → 2024 MONTH(Datum) → 6 DAY(Datum) → 1 DATEDIFF(d1,d2) → Differenz in Tagen
Häufige Abfragemuster (IHK-Aufgaben)
Top-N-Abfrage
-- Die 3 teuersten Bestellungen SELECT * FROM Bestellung ORDER BY Preis DESC LIMIT 3;
Duplikate finden
SELECT Produkt, COUNT(*) AS Anzahl FROM Bestellung GROUP BY Produkt HAVING COUNT(*) > 1;
Bedingte Berechnung
-- Gesamtwert je Zeile berechnen SELECT Kunde, Produkt, Preis * Menge AS Gesamtwert FROM Bestellung WHERE Preis * Menge > 200 ORDER BY Gesamtwert DESC;

Was ist SQL? — Definition & Einordnung
SQL — Structured Query Language
SQL ist eine standardisierte Datenbankabfragesprache zur Definition, Manipulation und Kontrolle von Daten in relationalen Datenbanksystemen (RDBMS). SQL ist kein Programm, sondern eine deklarative Sprache — man beschreibt was man will, nicht wie es berechnet wird.
Standard: ISO/IEC 9075 · Aktuell: SQL:2023
Relationales Datenbanksystem (RDBMS)
Ein RDBMS speichert Daten in Tabellen (Relationen), die über Schlüssel miteinander verknüpft sind. Beziehungen zwischen Tabellen werden durch Fremdschlüssel hergestellt.
Beispiele: MySQL, PostgreSQL, MariaDB, Oracle, MS SQL Server, SQLite
Relation = Tabelle
Eine Relation besteht aus:
· Attributen = Spalten (Name, Typ, Constraint)
· Tupeln = Zeilen (ein Datensatz)
· Domäne = Wertebereich eines Attributs
Eselsbrücke: SQL-Grundbegriffe
Tabelle = Relation

Spalte  = Attribut
Wert   = Domäne (Wertebereich)
→ „ZTSAW → RTAD"

Schlüsselkonzepte — Primär- & Fremdschlüssel
Primärschlüssel (PRIMARY KEY)
Ein Primärschlüssel identifiziert jeden Datensatz in einer Tabelle eindeutig. Er darf nicht NULL sein und muss in der gesamten Tabelle einmalig sein.
Kann aus einer oder mehreren Spalten bestehen (zusammengesetzter PK).
Fremdschlüssel (FOREIGN KEY)
Ein Fremdschlüssel ist ein Attribut, das auf den Primärschlüssel einer anderen Tabelle verweist. Er stellt die referentielle Integrität sicher — es darf nur auf existierende Werte verwiesen werden.
Kandidatenschlüssel
Alle Attribute oder Attributkombinationen, die als Primärschlüssel geeignet wären (eindeutig + minimal). Der gewählte wird Primärschlüssel, die restlichen heißen alternative Schlüssel.
Eselsbrücke: Schlüsselarten
„Primär Identifiziert Fremd Verweist"
PK = eindeutig, nicht NULL, pro Tabelle 1×
FK = zeigt auf PK einer anderen Tabelle

Normalisierung — 1NF · 2NF · 3NF
Definition Normalisierung
Die Normalisierung ist ein schrittweiser Prozess zur Strukturierung einer Datenbank, um Redundanzen (doppelte Daten) und Anomalien (Einfüge-, Änderungs-, Löschfehler) zu vermeiden.
1
1. Normalform (1NF)
Alle Attributwerte sind atomar (unteilbar). Keine Mehrfachwerte in einer Zelle. Jede Zeile ist eindeutig identifizierbar.
Verletzung: Zelle enthält „Müller, Meier, Schmidt"
2
2. Normalform (2NF)
1NF erfüllt + jedes Nicht-Schlüsselattribut ist voll funktional abhängig vom gesamten Primärschlüssel (keine Teilabhängigkeiten).
Nur relevant bei zusammengesetztem Primärschlüssel!
3
3. Normalform (3NF)
2NF erfüllt + kein Nicht-Schlüsselattribut ist transitiv abhängig von einem anderen Nicht-Schlüsselattribut.
Verletzung: PLZ → Ort (Ort hängt von PLZ ab, nicht vom PK)
Eselsbrücke: Normalformen
„Alles Voll Direkt"
1NF: Atom — keine Mehrfachwerte
2NF: Voll abhängig vom ganzen PK
3NF: Direkt abhängig — keine Transitivität
Anomalien (Warum normalisieren?)
Einfügeanomalie Daten können nicht gespeichert werden ohne andere Daten
Änderungsanomalie Gleiche Info an mehreren Stellen → inkonsistent
Löschanomalie Löschen eines Datensatzes verliert andere Infos

Transaktionen & ACID (TCL)
Transaktion
Eine Transaktion ist eine Folge von SQL-Befehlen, die als eine unteilbare Einheit ausgeführt wird. Entweder werden alle Befehle erfolgreich ausgeführt (COMMIT) oder alle werden rückgängig gemacht (ROLLBACK).
Klassisches Beispiel: Banküberweisung — Abbuchung + Gutschrift müssen zusammen gelingen.
ACID-Eigenschaften
Atomicity — ganz oder gar nicht
Consistency — DB bleibt konsistent
Isolation — Transaktionen stören sich nicht
Durability — nach COMMIT dauerhaft gespeichert
Eselsbrücke: ACID
„Alle Container Isoliert Dauerhaft"
Atomicity · Consistency · Isolation · Durability
BEGIN; UPDATE Konto SET Guthaben = Guthaben - 100 WHERE ID = 1; UPDATE Konto SET Guthaben = Guthaben + 100 WHERE ID = 2; COMMIT; -- beide oder keine

Merksätze & Kurzreferenz
Goldene Regeln für die IHK
· DDL = Struktur · DML = Daten · DQL = Abfragen
· PK = eindeutig + NOT NULL · FK = Verweis auf PK
· n:m → immer Zwischentabelle
· 1NF: atomar · 2NF: voll abhängig · 3NF: direkt abhängig
· ACID: Transaktion ist unteilbar
· NULL ≠ 0 ≠ '' — NULL bedeutet „unbekannt"
Eselsbrücke: NULL verstehen
NULL ist kein Wert — es ist Unwissen.
NULL = 0 → FALSCH (0 ist eine Zahl!)
Prüfen: immer mit IS NULL, nicht mit = NULL
